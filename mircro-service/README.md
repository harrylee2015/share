# 微服务

## 背景
  
* 互联网行业的快速发展

* 敏捷开发，快算迭代深入人心

* 容器技术的成熟


## 微服务的特征

 * 单一职责
 
 * 轻量级通讯
 
 * 隔离性
 
 * 有自己的数据
 
 * 技术多样性
 
 ## 微服务的优势
 
 * 独立性
 
 * 敏捷性
 
 * 技术栈灵活
 
 * 高效灵活
 
 ## 微服务的不足
 
 * 额外的工作（服务的拆分）
 
 * 数据一致性 （多个数据库）
 
 * 沟通成本 
 
 ## 微服务架构引入的问题
 
 * 微服务间如何通讯
 
 * 微服务如何发现彼此
 
 * 微服务怎样部署，更新，扩容
